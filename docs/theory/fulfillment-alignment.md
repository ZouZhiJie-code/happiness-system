# Fulfillment 维度对齐基线

最后更新：`2026-05-01`

## 理论背景

- 原始理论 PDF：`docs/theory/精简-如何实现幸福.pdf`
- 本文用于把理论里的“充实日志”翻译成当前产品和 AI 访谈要达成的技术定义。
- 截至 `2026-05-01`，当前代码已经按本文完成 fulfillment 的理论对齐、提问策略、完成标准、正文成稿、质量门、fallback、标题治理和用户边界收束。

## 1. 理论目标

- `fulfillment / 充实` 的核心任务，不是记录“今天很忙”，也不是记录“今天挺开心”。
- 它真正要确认的是：今天哪件事让我觉得这一天没有白过。
- 这种“没有白过”的感觉，可能来自三种主来源：
  - 事情真的往前推进了
  - 我在某个方向上真实积累了一点
  - 我的投入对他人或整体事情真的产生了作用
- 理论里写充实，不是为了给一天打鸡血式高分，而是为了减轻“我是不是又虚度了一天”的焦虑。
- 从长期看，充实会帮助用户确认什么样的努力对自己来说是有价值的；这些充实会慢慢积累成能力，能力再进一步转成价值。
- 因此，AI 在这个维度里最后要帮用户沉淀的，不是一个情绪标签，而是一条可信的“值得感标准”。
- 这里的“值得感标准”，也可理解为“重视线索”：什么样的投入、推进、积累或贡献，会让我觉得自己的力气花得值。

## 2. 最终日志状态

一篇合格的 fulfillment 日志，读完后应该让人知道：

- 今天哪件具体事情让我觉得充实。
- 这件事到底推进了什么、积累了什么，或者帮助了什么。
- 为什么它不是单纯的忙碌感，而是真正有分量。
- 这份分量更像来自完成、投入，还是贡献。
- 如果材料足够，读完后还能隐约感觉到：我重视什么样的努力，才会觉得这一天算数。

这篇日志的核心，不是“我做了很多事”，而是“我终于说清楚，什么样的投入对我来说不算白费”。

成稿读感应介于私密日记和复盘记录之间：

- 它可以有整理后的清晰度。
- 但不能像工作汇报、日报、周报或绩效总结。
- 用户读起来应该觉得“这是我整理过后的日志”，而不是“系统在替我总结效率”。

## 3. 必填槽位

没有下面这些槽位，就不能认为一篇 fulfillment 日志已经成立：

- `experience`
  - 真实发生过的具体片段。
  - 必须能落到一个事件、动作或情境，不能只是“今天挺充实”“今天学到了很多”这种总括。
- `progressEvidence`
  - “今天没有白过”的证据。
  - 它回答的是：到底推进了什么、完成了什么、练到了什么、积起来了什么，或者帮到了什么。
  - 这是 fulfillment 的核心槽位；没有它，就只有忙碌感，没有充实感。
- `valueSignal`
  - fulfillment 的高阶槽位，产品语义固定为“值得感标准”。
  - 它回答的是：什么样的投入，会让我觉得这一天算数。
  - 它不是抽象价值观口号，而是从具体证据里长出来的一条判断。

这三个槽位会直接影响：

- 访谈推进
- 是否允许生成日志
- 正文最后能不能自然收束到“我重视什么样的努力”

## 4. 可选槽位

这些槽位会提升日志质量，但不是每次都必须有：

- `feeling`
  - 当时更偏踏实、安心、笃定、进入状态，还是松一口气。
  - 它能帮助日志更像“我当时的感受”，但不能代替进展证据。
- `fulfillmentType`
  - 当前理论上固定为三类：
    - `推进完成型`
    - `投入积累型`
    - `协作贡献型`
  - 这个类型只用于帮助系统组织理解和后续成稿，不应代替实际理解。

可选槽位的使用原则：

- 可以帮助 AI 判断应该怎么追问、怎么组织正文。
- 不能因为想“写完整”就硬补。
- 如果证据不足，宁可不写，也不要把普通忙碌硬抬成某种充实类型。

## 5. 完整完成标准

默认完整模式成立，需要同时满足：

- 已找到可信 `experience`
- 已说清可信 `progressEvidence`
- 已形成可信 `valueSignal`

这里的“可信 `valueSignal`”必须满足：

- 它能从前面的片段和证据里自然推出。
- 它说的是“什么样的努力让我觉得值得”，不是“我这个人价值观很正确”。
- 它的力度只到“值得感标准”为止，不默认上升成长期人生方向、职业使命或稳定人格判断。

完整模式下，结尾允许轻轻收束出这条线索，例如：

- 我会对真正往前推进的感觉更有分量。
- 比起表面忙，我更看重这种一点点积起来的扎实感。
- 让我踏实的，还是自己做的事确实对别人有用。

但完整模式仍然禁止：

- 写成方法论条目
- 写成自我激励口号
- 写成“从此以后我就知道了人生答案”

## 6. 部分完成标准

如果用户明确不想继续深挖，或者当前材料只够到半程，只要满足下面条件，就允许生成“当前版本日志”：

- 已找到可信 `experience`
- 已说清可信 `progressEvidence`

部分模式的本质是：

- 已经足够确认“今天不是空转”
- 但还不够稳，不能假装已经形成清晰的值得感标准

部分模式下允许写：

- 这件事为什么算数
- 为什么它让我踏实
- 我现在更明白，自己会对什么样的推进或积累有分量感

部分模式下明确禁止：

- 把还没说透的感觉写成稳定规律
- 硬写 `valueSignal`
- 写成“我已经找到真正方向”
- 写成“只要我做什么，我就一定会觉得值得”的公式句

如果用户明确拒绝继续追问，但还没有可信 `progressEvidence`：

- 不能继续追工作细节。
- 不能硬生成日志。
- 应进入低压选择，让用户决定“只补一句 / 换一个片段 / 先退出”。
- “只补一句”只要求补今天为什么不算白过，不再追方案细节或完整步骤。

## 7. 写作控制层

### 标题控制

- 标题上限仍为 `16` 字。
- 标题必须是总结型短短语，优先从 `progressEvidence / valueSignal / fulfillmentType` 提炼。
- 典型方向包括“从结构到落地”“主线终于理顺”“让协作接上”这类语义压缩。
- AI 返回的长事件句、流水句、字段句或截断句不能直接展示。
- fallback draft 也必须使用同一套标题生成策略。

fulfillment 的正文生成应沿用统一写作控制接口，但其理论目标要固定如下：

- `voiceMode: "journal"`
  - 读起来像用户自己写下来的整理，而不是外部评价。
- `narrativeOrder: "scene_core_shift_close"`
  - 正文顺序默认是：
    - 从具体片段进入
    - 写清这件事为什么不算白过
    - 写清它为什么有分量
    - 最后轻收
- `closingMode`
  - 当前代码复用统一类型：
    - `complete -> "stable_clue"`
    - `partial -> "current_understanding"`
  - 在 fulfillment 语义里分别表示“轻收值得感标准”和“只停在当前不算白过的理解”。
- `toneBanSet`
  - 周报腔
  - 汇报腔
  - 系统解释腔
  - 正确废话式成长口号
  - 建议腔

具体到 fulfillment，这套控制要确保：

- 开头不能从“今天很充实”这种抽象判断起笔。
- 中间不能只列完成事项，而不写这些事情为什么有分量。
- 结尾不能写成自我管理建议，不能写成“以后要继续保持”。

## 8. 质量门规则

fulfillment 至少要拒收以下问题：

- `missing_scene_anchor`
  - 全文没有可信具体片段，只有抽象总结。
- `busy_without_progress`
  - 只有忙碌和任务堆积，没有“今天没有白过”的证据。
- `progress_slogan_tone`
  - 用“成长、进步、提升”这种大词替代真实推进细节。
- `forced_value_signal`
  - 证据不足却硬写“我重视……”或“我在意……”。
- `report_tone`
  - 写成日报、周报、绩效总结、项目汇报。
- `fake_direction_escalation`
  - 从一次局部充实直接跳到人生方向、职业使命或宏大价值判断。
- `partial_fake_rule`
  - 在部分模式下写出稳定规律或强结论。

其中最关键的产品禁令是：

- 不能像汇报
- 不能像效率工具自动总结
- 不能只有忙，没有算数的证据
- 不能没有证据就硬提炼“值得感标准”

## 9. fallback 原则

fulfillment 的 fallback 也必须像日志，不像说明文，更不像工作复盘模板。

正文骨架固定为：

- 片段起笔
- 写清推进 / 积累 / 贡献证据
- 写清为什么今天算数
- 结尾轻收

完整模式 fallback：

- 可以自然收束到可信 `valueSignal`
- 结尾的力度只到“我更知道什么样的努力对我来说值得”
- 不能写成价值观口号、方法总结或未来承诺

部分模式 fallback：

- 只能停在“这件事为什么不算白过”
- 可以写“我现在更明白，自己会对这种推进更有分量感”
- 不能假装已经形成稳定的值得感标准

无论完整还是部分模式，都必须保持以下边界：

- `fulfillment` 不等于 `joy`
- `fulfillment` 不等于 `improvement`
- `fulfillment` 不等于高产
- 小推进、小练习、小协作都成立，不要求大成果
- 真正要确认的，不是“我今天做了多少”，而是“什么样的努力对我来说算数”
