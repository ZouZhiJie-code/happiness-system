# 维度正文生成模板

最后更新：`2026-05-05`

本文用于把“某个维度的日志正文应该怎么生成”沉淀成统一模板，避免后续每个维度都重新从 prompt 硬凑。

当前约束：
- 本模板不定义前端交互。
- 本模板不要求新增公开 API。
- 本模板不要求改数据库 schema。
- 本模板只约束“理论 -> 槽位 -> 完成标准 -> 写作控制 -> 质量门 -> fallback”的生成方法。
- 截至 `2026-05-01`，所有维度标题还必须经过统一语义短标题治理，不能用长事件句机械截断。

## 1. 使用方式

后续每个维度都按下面同一套章节补齐，不允许跳章节：

1. 理论目标
2. 最终日志状态
3. 必填槽位
4. 可选槽位
5. 完整完成标准
6. 部分完成标准
7. 写作控制层
8. 质量门规则
9. fallback 原则
10. 标题治理与用户边界

## 2. 模板定义

### 2.1 理论目标

要回答：
- 这个维度对应的理论任务是什么？
- 用户为什么要写这一类日志？
- AI 访谈最后要帮用户沉淀出什么状态，而不只是“聊过”这件事？

### 2.2 最终日志状态

要回答：
- 一篇合格日志读完后，读者应该知道什么？
- 这篇日志最核心要留下的，不是事件本身，而是什么理解？
- 这篇日志应该更像日记、复盘、说明书，还是介于其中的某种形态？

### 2.3 必填槽位

定义：
- 没有这些槽位，就不能认为这一篇日志已经成立。
- 必填槽位要直接影响访谈推进、生成时机和正文内容。

### 2.4 可选槽位

定义：
- 这些槽位会提升日志质量或长期价值，但不是每次都必须有。
- 可选槽位不能被硬写，也不能为了“完整”强行补问。

### 2.5 完整完成标准

定义：
- 在什么条件下，这个维度已经可以形成“完整日志”。
- 默认完整模式的结尾是否允许收束出稳定线索、规律、方向或其他高层结果。
- 这一层必须由服务层规则控制，不能只靠 prompt 感觉判断。

### 2.6 部分完成标准

定义：
- 如果用户明确不想继续深挖，或者当前证据只够到半程，什么条件下允许生成“当前版本日志”。
- 部分模式下哪些高层结论必须禁止写。

### 2.7 写作控制层

当前标准控制层接口：
- `voiceMode`
- `narrativeOrder`
- `closingMode`
- `toneBanSet`

要求：
- 写作控制层由 `draft-policies.ts` 统一生成。
- prompt 必须显式消费写作控制层，而不是只靠散落自然语言指令。
- 以后每个维度都可以扩展自己的 profile，但接口入口保持统一。

### 2.8 质量门规则

至少要覆盖：
- 结构词/字段词禁令
- 系统腔/解释腔禁令
- 缺少具体片段锚点时的拒收
- 部分模式下的假规律拒收
- 该维度特有的禁止写法
- 内部理论解释不能直接出现在用户可见正文或 fallback draft 里

原则：
- 质量门不是补 prompt 的重复描述。
- 质量门要能拦住“模型看似写了，但产品上不能给用户看”的稿子。

### 2.9 fallback 原则

要求：
- fallback 不能只是“最保守的说明文”。
- fallback 必须和 prompt 使用同一套正文骨架。
- 完整模式和部分模式的 fallback 必须分开设计，不能共用同一收尾模板。

### 2.10 标题治理与用户边界

标题治理要求：
- 标题上限保持 `16` 字。
- 标题必须是总结型短短语，不能像字段名、流程句或被截断的长句。
- AI draft 标题和 fallback draft 标题都必须经过同一套后端质量检查。

用户边界要求：
- 用户明确表示“不想继续 / 不要再追问 / 直接生成 / 总结日志 / 整理成日志 / 追问没有意义”时，边界优先级高于槽位完整度。
- 材料足够时允许 partial 当前版本日志。
- 材料不足时不能继续补槽位式追问，应提供“只补一句 / 换一个片段 / 先退出”。

## 3. Joy 标准样板

`joy` 是当前第一份完整样板，后续其他维度都按这份样板的方法落地。

### 3.1 理论目标

- 帮用户把真实发生过的开心片段沉淀成一条可复用的个人线索。
- 不只是复盘一件开心的事，还要让用户更知道自己会被什么打动、什么会让自己进入更好的状态。

### 3.2 最终日志状态

- 读完后，知道哪个片段让我开心。
- 读完后，知道真正让我有感觉的点是什么。
- 读完后，知道这份开心带来了什么状态变化，或满足了我什么在乎 / 需要。
- 如果材料足够，读完后还能隐约留下“我在什么情况下更容易进入好状态”的线索。
- 成稿读感介于私密日记和复盘记录之间，更像“我整理过后的表达”，不是系统分析。

### 3.3 必填槽位

- `joyMoment`
- `joySource`
- `stateShift`
- `meaningNeed`
- `manualClue`
- `delightSignature`

注：
- `meaning_track` 下，`stateShift` 和 `meaningNeed` 在正文层至少要自然成立一个，`manualClue` 是完整收尾门槛。
- `delight_track` 下，`stateShift` 必须成立，`delightSignature` 是完整收尾门槛。
- `delightSignature` 必须是可直接写入日志的自然中文线索，不能是抽象理论词、抽象短语或单纯状态词。

### 3.4 可选槽位

- `directionSignal`
- `valueImpact`
- `durability`
- `tags`

### 3.5 完整完成标准

- 已找到可信 `joyMoment`
- 已说清 `joySource`
- `meaning_track`：已拿到 `stateShift` 或 `meaningNeed` 至少一个，并沉淀出可信 `manualClue`
- `delight_track`：已拿到可信 `stateShift`，并沉淀出可信 `delightSignature`
- `delightSignature` 不能靠长度兜底放行；`象征意义 / 确定性 / 动作本身 / 简单性 / 启动信号 / 仪式感` 这类抽象词，以及 `清醒 / 从容 / 有准备` 这类单纯状态词，都不能让 `delight_track` 完整闭合
- 完整模式的结尾允许轻收束出这条线索，但不能写成公式、方法总结或操作说明

### 3.6 部分完成标准

- 用户明确不想继续提炼规律时，只要 `joyMoment + joySource + stateShift|meaningNeed` 已成立，就允许生成当前版本日志
- 部分模式只能停在当前发现
- 部分模式禁止写成稳定规律，禁止出现“只要……我就更容易……”这类结论句式

### 3.7 写作控制层

当前 `joy` 固定值：
- `voiceMode: "journal"`
- `narrativeOrder: "scene_core_shift_close"`
- `closingMode`
  - `complete -> "stable_clue"`
  - `user_override_partial -> "current_understanding"`
- `toneBanSet`
  - `"这次访谈"`
  - `"我已经整理出"`
  - `"使用说明书"`
  - `"当前版本日志"`
  - `"至少到现在我已经更清楚"`
  - `"总的来说"`
  - `"总结起来"`
  - `"以后可以"`
  - `"我应该"`

### 3.8 质量门规则

当前 `joy` 至少拒收以下问题：
- `structured_terms`
- `field_labels`
- `system_phrase`
- `system_tone`
- `summary_tone`
- `advice_tone`
- `missing_scene_anchor`
- `forced_manual_clue`
- `forced_delight_signature`
- `fake_rule_tone`
- `false_depth_escalation`
- `delight_shaming_tone`
- `track_mismatch`

其中最关键的产品禁令是：
- 不能像总结卡
- 不能像系统解释
- 不能全篇抽象没有片段
- 部分模式不能写出稳定规律

### 3.9 fallback 原则

- fallback 也必须像日志，不像系统摘要
- 骨架固定为：
  - 片段起笔
  - 真正开心点 / 核心感受
  - 状态变化或被满足的在乎
  - 结尾轻收
- 完整模式 fallback：
  - `meaning_track` 允许用自然体会轻收束 `manualClue`
  - `delight_track` 允许轻收束 `delightSignature`
  - 不能把内部 `theorySummary` 直接写成正文句子
- 部分模式 fallback：
  - 只能停在“我现在更知道什么会打动我 / 我更在意什么”
  - 不能写成稳定规律

### 3.10 Joy 标题与理论腔补充

- joy 标题不能使用 `一下被带轻 / 象征意义` 这类伪中文或理论词；早起、多出时间、准备感场景应收束成 `清醒地开始` 这类自然短标题。
- joy 正文和 fallback draft 不能出现“这份开心更像轻快乐”“关键不是深意义”“这种会把状态轻轻带起来的方式”这类内部理论解释。
