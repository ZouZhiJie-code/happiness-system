# Interview Question Clarity Badcase Pack

**Source account:** `珊瑚礁`  
**Entry date:** `2026-05-26`  
**Purpose:** 为后续提问架构改造、subagent 实现、code review 与回归测试提供统一案例语料。  
**Usage rule:** 除了基础设施任务外，所有涉及问法生成、repair、文案 gate、badcase 回归的 subagent 都应先阅读本文件。

---

## How To Use This Pack

每个 case 都包含：

- 原始上下文
- 实际坏问题
- 用户反应
- badcase 类型
- 为什么坏
- 更好的提问方向
- 验收提示

执行任务时不要只复述“为什么坏”，要直接对照：

1. 新链路是否避免生成同类问法
2. repair 是否真正降负担
3. 问题是否更像自然中文
4. 用户是否能直接回答，不需要先在脑内做抽象压缩

---

## Taxonomy

- `A1` 抽象名词先行
- `A2` 双重或三重认知动作
- `A3` 锚点漂移
- `A4` 伪降压 / 换壳重问
- `A5` 不存在的 mental object
- `A6` 合规但不像人说话
- `A7` 边界处理失败

## Case 1: Joy 漂移到颜色 / 材质

**Case ID:** `JOY-DRIFT-MATERIAL`  
**Dimension:** `joy`  
**Session:** `15dbf43a-8d7b-49e0-a040-fc3abaa22275`

### Context

用户起点非常清楚：

- 收到扎根工程赠礼
- 有仪式感
- 见证自己开始一段学习
- 心里空旷，身体松弛

### Transcript Slice

用户：我觉得这个是不太可复现的，因为他就是收到礼物那一刻比较开心嘛。可能复现的话，就也多给自己买这种礼物了，多搞点仪式感给生活。  
AI：当你给自己买礼物或搞仪式感时，什么样的礼物或仪式最容易让你进入那种空旷松弛的状态？  
用户：我觉得是有一些纪念意义的吧，或者是视觉感官上是比较好的，有质感的。比如花呀什么的  
AI：这种视觉或质感上的美感，具体是哪种颜色、形状或材质，最容易让你感到那种空旷松弛？

### Failure Type

- `A3` 锚点漂移
- `A6` 合规但不像人说话

### Why It Is Bad

- 用户原始核心是“学习起点被看见 + 仪式感 + 松弛”，不是审美偏好画像。
- AI 把“花、有纪念意义、有质感”继续推成颜色/形状/材质，已经偏离这次开心的主轴。
- 这类追问对日志成稿帮助小，对用户理解也不自然。

### Better Direction

优先继续问：

- 这份礼物为什么会让你觉得“这段学习真的开始了”
- 仪式感最打动你的那一下是什么
- 这种松弛更像“被确认了”还是“终于落地了”

不要继续问：

- 颜色
- 形状
- 材质

### Acceptance Note

修复后，`joy` 在类似场景下不应再把“有质感/有纪念意义”自动扩展成审美细节问法，除非用户主动把开心核心明确落在审美体验本身。

## Case 2: Joy 边界后继续追问

**Case ID:** `JOY-BOUNDARY-REPEAT`  
**Dimension:** `joy`  
**Session:** `15dbf43a-8d7b-49e0-a040-fc3abaa22275`

### Transcript Slice

AI：这种视觉或质感上的美感，具体是哪种颜色、形状或材质，最容易让你感到那种空旷松弛？  
用户：具体的就不说了吧  
AI：抛开具体的颜色或材质不谈，什么样的内容或场景节奏，最容易把你带进这种空旷松弛的状态？  
用户：结束本轮访谈  
AI：抛开具体的颜色或材质不谈，什么样的内容或场景节奏，最容易把你带进这种空旷松弛的状态？

### Failure Type

- `A4` 伪降压
- `A7` 边界处理失败

### Why It Is Bad

- “具体的就不说了吧”已经是明显边界信号，系统只做了表面换问法。
- “结束本轮访谈”是更明确的停止意图，系统却重复了上一题。
- 这是产品边界协议没有落实到最终用户问题上的典型 badcase。

### Better Direction

在第一次边界后：

- 要么收束到一句更低负担的问题
- 要么直接给 low-pressure choice

在“结束本轮访谈”后：

- 不应继续追问
- 应直接进入结束/choice/整理当前版本

### Acceptance Note

修复后，同类 case 不应出现“边界后换皮重问”或“明确停止后继续发原问题”。

## Case 3: Fulfillment 抽象价值问懵用户

**Case ID:** `FULFILLMENT-ABSTRACT-STANDARD`  
**Dimension:** `fulfillment`  
**Session:** `533f4bac-5fce-40ea-9d87-807d8ffaf6a9`

### Context

用户已经给了足够多的材料：

- 回顾过去经历
- 看到成长
- 重新梳理后学到新东西
- 因此坚定以后继续记录工作，便于复盘

### Transcript Slice

AI：这种“学到新东西”的充实感，让你觉得什么样的努力才算没白费？  
用户：……坚定了之后要继续地去记录自己的工作……后续复盘的时候能够学到很多新的东西……  
AI：回头看“看到了自己的成长”这层进展，什么样的投入会让你觉得自己的力气花得值？  
用户：不明白你问的什么意思  
AI：如果只留一句最算数的标准，你会怎么说？

### Failure Type

- `A1` 抽象名词先行
- `A4` 伪降压
- `A6` 合规但不像人说话

### Why It Is Bad

- 用户实际上已经回答了“什么样的努力算数”，AI 没有识别到可以收束。
- “什么样的投入会让你觉得自己的力气花得值”是典型 AI 腔。
- 用户听不懂后，repair 虽然变成了“如果只留一句最算数的标准”，但仍然要求用户先做抽象总结。

### Better Direction

更好的后续问题应该是：

- 所以对你来说，能留下可回看的记录，这件事本身就很算数，对吗？
- 如果只留一句，你最想记住的是“记录下来，后面才能看见新的东西”这一点吗？
- 刚才这些里，最让你觉得今天没白过的是哪一句？

### Acceptance Note

修复后，这类用户已经给出“值得感标准”雏形的场景，不应继续追问更高抽象层问题。

## Case 4: Reflection 连续追问不存在的“具体瞬间”

**Case ID:** `REFLECTION-NONEXISTENT-MENTAL-OBJECT`  
**Dimension:** `reflection`  
**Session:** `b512bed9-2b54-434b-91e4-d5e17e42cc28`

### Context

用户真正的内容是：

- 刷视频是一种逃避
- 更轻松，不用主动思考
- 背后可能有畏难情绪
- 对未知结果的焦虑
- 不知道具体该干什么时更容易逃避

### Transcript Slice

AI：当你意识到“无聊，不知道该干啥”时，心里会闪过什么念头或画面，才让你决定刷视频而不是做别的？  
用户：其实也不是闪过了什么念头，只是刷视频会更加的轻松……  
AI：以后再遇到类似情况，你会先看哪个更具体的反应或信号，提醒自己别只看“看起来合适”？  
用户：可能是因为最近做其他的事情都要思考怎么做，有点畏难情绪……  
AI：这种“畏难”的感觉，在刷视频之前，具体会以什么方式先让你察觉到？  
用户：搞一会就休息会……  
AI：今天上午“搞一会就休息会”的时候，心里具体在比较哪两个选项，才让休息看起来比继续搞更合适？  
用户：其实也不是在比较……

### Failure Type

- `A2` 双重认知动作
- `A5` 不存在的 mental object
- `A4` 伪降压

### Why It Is Bad

- AI 预设用户心里一定有“闪过的念头 / 比较的选项 / 可观察的信号”。
- 用户已经多次说明自己不是这样运作的，系统仍在追同一种对象。
- 这不是用户表达不够，而是问题设定错了。

### Better Direction

应该改成：

- 当时最明显的是想逃开，还是脑子不想动？
- 你刚说“刷视频更轻松”，这句对你来说是不是这次最关键的发现？
- 如果只留一个判断，你现在更像是在躲“要主动想下一步”这件事，对吗？

### Acceptance Note

修复后，reflection 不应反复预设用户存在“比较/念头/信号”这种对象；当用户明确否定后，问题必须换到用户已经说出来的真实语言上。

## Case 5: Reflection 问题多步叠加

**Case ID:** `REFLECTION-MULTI-ACTION`  
**Dimension:** `reflection`  
**Session:** `b512bed9-2b54-434b-91e4-d5e17e42cc28`

### Transcript Slice

AI：这种对未知结果的焦虑，具体在哪个瞬间让你觉得“搞”不下去了，而不是还能再坚持一小会儿？  
用户：因为也不知道具体该干什么的时候，就会有这种想法  
AI：这种“不知道具体该干什么”的模糊感，具体在哪个瞬间让你觉得刷视频比想清楚下一步更值得？

### Failure Type

- `A2` 双重或三重认知动作
- `A6` 合规但不像人说话

### Why It Is Bad

- 同一句里混合了回忆瞬间、比较两个选项、解释阈值。
- 用户必须先在脑内建模“坚持一小会儿”和“更值得”两个比较对象，负担过高。

### Better Direction

应该拆开成一步：

- 你刚才说“不知道该干什么”，这是不是最容易把你推去刷视频的那一下？
- 那一刻更明显的是焦虑，还是脑子不想再想了？

### Acceptance Note

修复后，类似问题必须能被一个具体短回答直接接住，而不是要求用户先内化 AI 的复杂句式。

## Case 6: Improvement 用户说“不知道”后仍然继续硬追

**Case ID:** `IMPROVEMENT-I-DONT-KNOW`  
**Dimension:** `improvement`  
**Session:** `ab9693bb-9b55-4c65-8704-5939bbbc66e2`

### Transcript Slice

AI：下次再遇到类似情况，你最想先试的那一步是什么？  
用户：我也不知道下面最想试的一步是什么，反正就是会有这样的迷茫期。  
AI：如果下次在早餐后再次感到“无聊、没兴奋感”，你最先能调整的一个小动作会是什么？

### Failure Type

- `A4` 伪降压
- `A5` 不存在的 mental object

### Why It Is Bad

- 用户已经明确说“不知道”，说明当前还没形成最小动作。
- AI 看似收窄到“早餐后、无聊、没兴奋感”，但本质仍在追问同一个对象：请给我一个动作。

### Better Direction

应该改成更低负担：

- 如果现在还说不出动作，那你至少知道自己卡在哪一步了吗？
- 说到这里，你更想先记住“早餐后容易空掉”这一点，还是“没兴奋感时容易散掉”这一点？
- 要不要先停在当前这版观察，不硬往“下次动作”上推？

### Acceptance Note

修复后，当用户明确表示“不知道下一步动作”时，系统不应立刻再追一个换皮动作题。

## Case 7: Gratitude 过早推到抽象“珍惜什么”

**Case ID:** `GRATITUDE-ABSTRACT-TREASURE`  
**Dimension:** `gratitude`  
**Session:** `87a7abb3-353b-4bd9-be69-c8e528357d51`

### Context

用户已经说得很具体：

- 邹邹陪伴
- 给面试建议
- 一针见血地分析焦虑和迷茫
- 感觉被破除迷茫和焦虑

### Transcript Slice

AI：这种能一针见血地帮你破除迷茫的回应，让你觉得这份感谢里，最值得珍惜的是什么？

### Failure Type

- `A1` 抽象名词先行
- `A5` 不存在的 mental object
- `A6` 合规但不像人说话

### Why It Is Bad

- 用户还在“对方做了什么、帮到了我什么”的层级。
- AI 直接上升到“最值得珍惜的是什么”，抽象跃迁过大。

### Better Direction

更自然的问法应该是：

- 他说的哪一句，最让你觉得自己不是一个人在乱撞？
- 比起陪你、玩游戏、给建议，今天最打动你的是哪一下？
- 你更想记住的是“有人陪着你”，还是“有人真的看懂你在焦虑什么”？

### Acceptance Note

修复后，gratitude 在用户只给出两轮信息时，不应过早上升到关系价值抽象总结。

## Case 8: 统一的设计结论

这些 case 共同说明：

1. 当前系统缺的不是理论目标，而是“人话翻译层”
2. `judgment_clue` 是最危险的抽象问题来源
3. repair 不能只换说法，必须降认知负担
4. final gate 不能只检查违规词，必须检查“用户能不能直接答”
