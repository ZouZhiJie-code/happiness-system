① 心智模型

把「文档装配机」换成「会生长的一天」

旧模型把会话、草稿、双栏、生成、保存这些实现细节全暴露给用户；新模型只暴露一个对象——你的今天。

改动前 · 文档装配

用户在「操作一台机器」

  • 在框里聊天 → 找按钮 → 生成草稿
  • 草稿落到一个抢空间的小窗里手动改
  • 逐个维度重复 5 次
  • 再进另一个工作区拼「总日志」
  • 用户要理解 session / 草稿 / 暂存 / 保存
→
改动后 · 会生长的一天

用户在「记录自己的一天」

  • 对话独占画布，专注表达
  • 日志随时从书签拉出，在专注书页里读写
  • 一条书脊显示五维 + 完整的真实进度
  • 「把今天收成」一步收束当天
  • 用户只需理解一件事：今天记到哪了
② 访谈画布

对话独占画布 · 书脊导航 · 书签是唯一日志入口

取消左右抢空间的双栏与溢出的工具栏。进度收敛成一条书脊；「整理成日志」永远在同一个位置——右侧书签丝带，不再在 header 与对话卡之间跳。

③ 日志书页

从抢空间的小窗，变成专注的书页浮层

点书签，日志从右侧滑出占满高度，对话在身后变暗。读写日志是一个「专注模式」，不是一根永远在抢宽度的柱子。保存状态显性、自动暂存有声。

④ 把今天收成

四步手动拼装 → 一次「收成今天」

完整日志不再是另一个工作区里的二次装配，而是书脊尽头一个收束动作：自动把有草稿的维度落库 → 汇编 → 保存，并透明展示纳入与跳过。

命名混用（总日志/汇总日志）；「实」有草稿没保存，汇总时被静默丢弃，用户毫不知情。

一次点击跑完整条链；命名统一「完整日志」；纳入与跳过一目了然，不再静默丢失草稿。
